git仓库步骤
1、 git init命令把这个目录变成Git可以管理的仓库;
2、git add . 把文件夹所有文件添加到仓库;
3、git commit -m “add all file” 提交至本地 仓库 -m 添加提交说明;
4、git remote add origin git@github.com:starfishing/learngit.git 关联远程仓库;
5、git push -u origin master 推送至远程仓库;
-u 指定推送远程分支，加了参数-u后，以后即可直接用git push 代替git push origin master
-f 将本地历史覆盖到远端仓库,强制推送。

补充:

1、git status 查看本地文件与远程分支是否存在差异;
2、git diff 查看修改的内容
3、git branch 查看当前使用分支；
-a 查看所有分支;
4、git checkout “XXX” 切换分支
git branch “XXX” 创建分支
创建并切换分支：git checkout -b mybranch
5、删除本地分支 Chapater6
git branch -d Chapater6
6、git pull origin master 从master上拉取分支

7、merge “XX分支” 把XX分支合并到当前分支
8、git remote add origin git://127.0.0.1/abc.git 这样就增加了远程仓库abc。
9、git remote remove origin移除远端仓库

提交到git时，忽略部分IDE产生的文件
在根目录下创建.gitignore文件， 注意：新加.gitignore只能忽略那些原来没有被提交过的文件，
如果某些文件已经被纳入了版本管理中，则修改.gitignore是无效的。
解决方法就是先把本地缓存删除（改变成未track状态），然后再提交:
git rm -r –cached .
git add .
git commit -m ‘update .gitignore’

gitignore文件内容，举例如下：

//target
//.project
//.classpath
//.settings
